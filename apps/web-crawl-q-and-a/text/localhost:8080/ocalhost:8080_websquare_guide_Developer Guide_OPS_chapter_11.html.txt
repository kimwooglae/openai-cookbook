
웹 표준 개발 가이드11.웹 표준 개발 가이드11.1개요웹스퀘어를 이용한 프로젝트에서 멀티브라우저를 지원하기 위해서는 다음과 같은 방법으로 프로그래밍을 작성해야 합니다.웹스퀘어 API 문서에서 제공하는 기능을 사용합니다. ECMAScript3.0 sepc 에 정의된 API 를 사용하여야 합니다. 웹스퀘어에서 사용을 금지한 방법을 사용하지 않는다. 1,2,3 번 항목을 제외한 나머지는 일반적으로 알려진 웹표준을 준수하는 프로그래밍 방법을 사용합니다.웹스퀘어 API문서에서 제공하는 기능을 사용하여 작성하되 ECMAScript3.0에 정의된 API 만을 사용하여야 합니다. (ECMAScript 에 대한 자세한 내용은 ECMA 을 참고합니다.)단, 웹스퀘어에서 사용을 금지하는 방법은 사용해서는 안됩니다.이 문서에서는 3번 웹스퀘어에서 사용을 금지하는 항목에 대하여 설명합니다.11.2웹스퀘어에서 금지하는 사항웹스퀘어에서 금지하는 사항은 아래와 같다. 만일 아래 항목 중 프로그램 구현을 위하여 꼭 필요한 경우에는 웹스퀘어 지원 인력에게 확인을 받아야 합니다.브라우저 객체(DOM)를 직접 제어하는 모든 API는 사용하면 안됩니다. 예를 들어 웹스퀘어에서 제공된 컴포넌트로 그려진 객체(DOM)에 대한 getElementById(), createElement()를 사용해선 안됩니다. document.write 와 같은 function 은 사용해선 안됩니다. 브라우저 Event 를 직접 제어하면 안됩니다. XMLHttpRequest를 직접 생성하고 조작하는 API 는 사용하면 안됩니다. WebSquare 에서 제공하는 submission 을 이용하여야 합니다. API 를 이용하는 경우 WebSquare.ModelUtil.executeSubmission()을 사용하여야 합니다. prototype.js나 jquery.js와 같은 오픈소스 JavaScript Framework를 동시에 사용하면 안됩니다. prototype 확장은 하지 말아야 합니다.11.3브라우저에 따른 ECMAScript 지원 여부11.3.1ECMAScript Language constructsFeatureIE6IE7Firefox2Firefox3Opera9TypesUndefinedYYYYYNullYYYYYBooleanYYYYYStringYYYYYNumberYYYYYObjectYYYYYBasic expressionsthisYYYYYnewYYYYYPostfix ++YYYYYPostfix --YYYYY,IIYYYUnary expressionsdeleteYYYYYvoidYYYYYtypeofYYYYYprefix ++YYYYYPrefix --YYYYY+YYYYY-YYYYY~YYYYY!YYYYYMath expressions+YYYYY-YYYYY*YYYYY/YYYYY%YYYYYRelational expressions==YYYYY!=YYYYY===YYYYY!==YYYYY<YYYYY>YYYYY<=YYYYY>=YYYYYInstanceofYYYYYinYYYYYBinary expressions&YYYYY^YYYYY|YYYYY&&YYYYY||YYYYY<<YYYYY>>YYYYY>>>YYYYYAssignment expressions=YYYYY*=YYYYY/=YYYYY%=YYYYY+=YYYYY-=YYYYY<<=YYYYY>>=YYYYY>>>=YYYYY&=YYYYY^=YYYYYStatementsExp ?
